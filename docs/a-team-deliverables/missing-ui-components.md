## Implementation Notes

### Availability Display
- Start with calendar grid (most familiar pattern)
- Can A/B test with list view later
- Consider "availability preview" in search results (just dots)

### Time Selection
- Keep it inline after calendar selection
- One-tap from available time
- Show duration options only if instructor offers multiple

### Search Results
- Show photo for trust
- 3 next available times maximum
- Quick book vs. view profile options
- Mobile: Cards stack vertically

### Booking Form
- Collect minimum required info
- Autofill for returning users
- Payment at booking (not account creation)
- Clear total and policies
